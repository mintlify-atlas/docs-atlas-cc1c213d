Yarn is a set of open, unencumbered Minecraft mappings published by FabricMC under the Creative Commons Zero license. It replaces Minecraft's obfuscated class names (like `class_4970`) with meaningful names (like `AbstractBlock`), making it possible to write Minecraft mods with readable, maintainable code.

## What Yarn provides

Minecraft's code is shipped obfuscated — every class, method, and field has a meaningless name like `class_1234` or `method_5678`. Yarn maps those names to descriptive identifiers such as `BlockEntity`, `tick`, and `getWorld`, giving the Minecraft modding community a shared vocabulary.

## How Yarn works

Yarn sits in the middle of a three-namespace mapping chain: **official** (Mojang's obfuscated names) → **intermediary** (stable numeric names that don't change between Minecraft versions) → **named** (Yarn's human-readable names).

When you write a Fabric mod, Fabric Loom remaps your code through this chain so your mod works against the correct obfuscated names at runtime, while you write code against Yarn's readable names at development time.

<Steps>
  <Step title="Fork and clone the repository">
    Fork [FabricMC/yarn](https://github.com/FabricMC/yarn) on GitHub and clone your fork locally. You need Java 17 or higher installed.
  </Step>
  <Step title="Open Enigma">
    Run `./gradlew yarn` to launch the Enigma GUI. It will download the Minecraft jar, remap it through intermediary, and open it alongside the current mappings.
  </Step>
  <Step title="Edit mappings">
    Find unmapped or incorrectly named classes, methods, or fields in Enigma and assign better names. Consult the [naming conventions](/conventions/general) before you write anything.
  </Step>
  <Step title="Submit a pull request">
    Commit your changes, push to your fork, and open a pull request against the Yarn repository. See the [contributing guide](/contributing/submitting-mappings) for review expectations.
  </Step>
</Steps>
