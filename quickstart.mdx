This guide walks you through the full workflow for contributing a mapping to Yarn — from cloning the repository to opening a pull request. You do not need prior experience with Minecraft internals; you only need a working Java installation and a GitHub account.

<Warning>
  Copying names from MCP, Spigot, or Mojang's official obfuscation maps is **strictly forbidden**, even for inspiration. Yarn is CC0-licensed; importing names from more restrictive sources would compromise that. When in doubt, consult other contributors via a pull request or the FabricMC community spaces.
</Warning>

## Prerequisites

Before you start, make sure you have the following installed:

- **Java 17 or higher** — required to run the Yarn build scripts and Enigma.
- **Git** — for cloning the repository and managing your changes.
- A **GitHub account** — to fork the repository and open pull requests.

You can verify your Java version by running:

```bash
java -version
```

The output should show version 17 or higher.

## Contribute a mapping

<Steps>
  <Step title="Fork and clone the repository">
    Go to [github.com/FabricMC/yarn](https://github.com/FabricMC/yarn) and click **Fork** to create your own copy of the repository.

    Then clone your fork locally:

    ```bash
    git clone https://github.com/<your-username>/yarn.git
    cd yarn
    ```

    <Info>
      The active Minecraft version is indicated by the branch name. Make sure you are on the branch that matches the version you want to contribute to.
    </Info>
  </Step>

  <Step title="Open Enigma">
    Run the `yarn` Gradle task to download Enigma and the Minecraft jar, then launch the Enigma GUI automatically configured for the current mappings:

    <CodeGroup>

    ```bash Linux and macOS
    ./gradlew yarn
    ```

    ```bat Windows
    gradlew yarn
    ```

    </CodeGroup>

    Gradle downloads the Minecraft client and server jars, merges them, remaps through intermediary, and opens Enigma with the `mappings/` directory loaded. This may take several minutes on first run.

    <Tip>
      If you only want to work on classes that exist in both the client and server, run `./gradlew yarnCommon` instead. It opens Enigma against the server-only jar.
    </Tip>
  </Step>

  <Step title="Find an unmapped class or method">
    In Enigma's left-hand class tree, unmapped entries are displayed with their intermediary name (e.g. `class_4970`). Named entries show their human-readable Yarn name.

    Browse the tree or use **Search** to locate a class or method you want to name. Select it to see its decompiled source in the main panel.

    Read the [naming conventions](/conventions/general) before assigning any names. Yarn has specific rules around capitalisation, abbreviations, and terminology that all contributions must follow.
  </Step>

  <Step title="Add a name">
    Double-click the class, method, or field entry in the Enigma tree to open the rename dialog. Type the new name and press **Enter**.

    Enigma writes your change directly to the `mappings/` directory as an Enigma-format `.mapping` file. No additional save step is needed.

    If you want to add a Javadoc comment to document the mapped symbol, use **Edit → Edit Javadoc** in the Enigma menu.
  </Step>

  <Step title="Validate your changes">
    Before committing, run the build to check that your mappings are well-formed and consistent:

    <CodeGroup>

    ```bash Linux and macOS
    ./gradlew build
    ```

    ```bat Windows
    gradlew build
    ```

    </CodeGroup>

    The `build` task compiles a GZip-compressed Tiny mapping archive and runs several validation checks, including `checkMappings`, `checkUnpickDefinitions`, and a Javadoc lint pass. Fix any reported errors before proceeding.
  </Step>

  <Step title="Commit and push">
    Stage and commit your changes:

    ```bash
    git add mappings/
    git commit -m "Map <ClassName>: brief description of what it does"
    git push origin HEAD
    ```

    Use a concise commit message that describes what you named and why. Reviewers read commit messages during the pull request review.
  </Step>

  <Step title="Open a pull request">
    Go to your fork on GitHub and click **Compare & pull request**. Target the correct Minecraft version branch on `FabricMC/yarn`.

    In the pull request description, explain what you named and the reasoning behind the names you chose. Reviewers may suggest changes — respond to feedback and push additional commits to the same branch.

    Once approved, a maintainer will merge your contribution.
  </Step>
</Steps>
