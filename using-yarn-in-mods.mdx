---
title: "Using Yarn mappings in your Fabric mod"
description: "Add Yarn as a Gradle dependency via Fabric Loom so your mod compiles against human-readable Minecraft names instead of obfuscated identifiers."
---

Yarn is the set of name mappings that lets you write Fabric mods against readable class and method names instead of Mojang's obfuscated identifiers. As a mod developer, you consume Yarn through the [Fabric Loom](https://github.com/FabricMC/fabric-loom) Gradle plugin — you do not need to clone or build the Yarn repository yourself.

<Info>
  Java 17 or higher is required to build Fabric mods with Yarn. Make sure your local JDK and your Gradle toolchain are both on Java 17+.
</Info>

## Finding the latest Yarn version

Yarn versions follow the format `<minecraft_version>+build.<build_number>`. For example, `1.21.11+build.3` is build 3 of the Yarn mappings for Minecraft 1.21.11.

Visit [fabricmc.net/develop](https://fabricmc.net/develop/) to find the latest Yarn version for the Minecraft release you are targeting. The page lists the recommended dependency snippets for `gradle.properties` and `build.gradle`, updated whenever a new Yarn build is published.

Yarn artifacts are hosted on `maven.fabricmc.net` under the group `net.fabricmc` and artifact ID `yarn`.

## Adding Yarn to your project

Fabric Loom reads the Yarn version from a `minecraft_mappings` property and uses it to remap the Minecraft jar for compilation. The standard project layout separates version strings into `gradle.properties` and references them from `build.gradle`.

<CodeGroup>

```properties gradle.properties
minecraft_version=1.21.11
yarn_mappings=1.21.11+build.1
loader_version=0.16.10
```

```groovy build.gradle
plugins {
    id 'fabric-loom' version '1.12.0-alpha.39'
    id 'maven-publish'
}

dependencies {
    // Minecraft
    minecraft "com.mojang:minecraft:${project.minecraft_version}"

    // Yarn mappings — the :v2 classifier selects the Tiny v2 artifact
    mappings "net.fabricmc:yarn:${project.yarn_mappings}:v2"

    // Fabric Loader
    modImplementation "net.fabricmc:fabric-loader:${project.loader_version}"
}
```

</CodeGroup>

The `:v2` classifier at the end of the `mappings` dependency string selects the Tiny v2 mapping artifact. Always include it — the unclassified artifact uses the older Tiny v1 format and is not supported by current versions of Loom.

<Note>
  The Yarn version must match the Minecraft version your project targets. Using `1.21.11+build.1` mappings with `minecraft "com.mojang:minecraft:1.20.4"` will cause Loom to fail with a version mismatch error.
</Note>

## What Loom does with Yarn

When you run any Gradle task in your mod project, Loom:

1. Downloads the Minecraft jar for the version specified in `minecraft_version`.
2. Downloads the Yarn artifact for the version specified in `yarn_mappings`.
3. Remaps the Minecraft jar from official (obfuscated) names → intermediary → named, producing a deobfuscated jar that your code compiles against.
4. At build time, remaps your compiled mod back from named → intermediary → official so it works against the unmodified Minecraft runtime.

Your source code always uses Yarn's human-readable names. The remapping steps are transparent.

## Generating a deobfuscated Minecraft jar

If you want to inspect or decompile the Minecraft source with Yarn names applied — for example to pass to a decompiler or to explore the codebase — run:

<CodeGroup>

```bash Linux and macOS
./gradlew mapNamedJar
```

```bat Windows
gradlew mapNamedJar
```

</CodeGroup>

This produces a jar named `<minecraft_version>-named.jar` in the `build/` directory. Any symbols that Yarn has not yet mapped will fall back to their intermediary names (e.g. `method_1234`).

<Tip>
  The `mapNamedJar` output is useful for exploring the Minecraft codebase with a decompiler such as IntelliJ IDEA's built-in decompiler or CFR. It is not designed to be recompiled.
</Tip>

## IDE integration via the Javadoc jar

Yarn publishes a `javadoc` classifier artifact alongside each mapping release. It contains full Javadoc for the deobfuscated Minecraft API, generated by MappingPoet from Yarn's Javadoc comments. Your IDE can display these inline as you write mod code.

Fabric Loom automatically attaches the Javadoc jar when you use the `v2` mappings artifact — no extra configuration is needed. If your IDE is not showing Javadoc, verify that your project uses Fabric Loom and that the Yarn artifact resolves correctly.

The Javadoc jar artifact is available at:

```
net.fabricmc:yarn:1.21.11+build.1:javadoc
```

Visit [fabricmc.net/develop](https://fabricmc.net/develop/) to find the latest version string for the Minecraft release you are targeting.
